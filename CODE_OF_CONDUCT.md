# Code of Conduct

## Our Pledge

Flow is an open-source project by APEX Studio.

We are committed to maintaining a respectful, inclusive, and professional environment for everyone who participates in the Flow project.

## Our Standards

Examples of positive behavior include:

* Being respectful and constructive.
* Giving and accepting feedback professionally.
* Focusing on what is best for the project and its users.
* Communicating clearly and patiently.
* Respecting different viewpoints, experiences, and skill levels.

Examples of unacceptable behavior include:

* Harassment, intimidation, threats, or personal attacks.
* Discriminatory, insulting, or derogatory comments.
* Trolling, disruptive behavior, or repeated off-topic arguments.
* Sharing private information without permission.
* Publishing personal finance data, private backup files, credentials, signing keys, or other sensitive material.
* Any conduct that would reasonably be considered inappropriate in a professional open-source project.

## Scope

This Code of Conduct applies to participation in the Flow project, including:

* GitHub issues.
* Pull requests.
* Discussions.
* Code reviews.
* Documentation contributions.
* Other project-related communication spaces controlled by the maintainer or APEX Studio.

It also applies when someone is officially representing Flow or APEX Studio in a project-related context.

## Enforcement

The maintainer is responsible for clarifying and enforcing this Code of Conduct.

The maintainer may take appropriate action when behavior does not meet these standards, including:

* Requesting clarification or correction.
* Editing or removing inappropriate comments or contributions.
* Closing or locking issues or pull requests.
* Temporarily or permanently limiting participation in the project.

Enforcement decisions will be made with the goal of protecting the project, its users, and its contributors.

## Reporting

Please do not use public issues for private conduct reports or sensitive personal information.

Conduct concerns should be reported through the official APEX Studio GitHub organization or the repository maintainer profile.

Reports should include enough context to understand the concern, but should not include unnecessary personal data, private finance data, backup files, passwords, signing materials, or other sensitive information.

## Attribution

This Code of Conduct is inspired by common open-source community standards, including the Contributor Covenant.
